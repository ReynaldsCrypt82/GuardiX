## Layout Contract

### Split-Screen Auth Layout (source: D-13 in 01-CONTEXT.md)

All auth screens use a 50/50 split-screen layout at `lg` breakpoint and above:

| Panel | Width (lg+) | Width (below lg) | Content |
|-------|------------|-----------------|---------|
| Left (brand panel) | 50% | hidden | Gradient + product tagline + visual |
| Right (form panel) | 50% | 100% | Form content |

**Left panel specification:**
- Background: linear gradient from `hsl(221 83% 53%)` (blue-600) to `hsl(262 83% 58%)` (violet-600)
- Overlay: `bg-black/10` to deepen gradient
- Content: product logo top-left, tagline centered, decorative elements (abstract card shapes)
- Tagline copy (pt-BR): "Gestão completa para corretoras de seguros e consórcio."
- Sub-tagline copy: "Do cadastro ao fluxo de caixa — tudo em um lugar."

**Right panel specification:**
- Background: `bg-background` (white in light mode)
- Padding: 48px horizontal, 40px vertical (`px-12 py-10`)
- Max-width of form container inside right panel: 400px, centered
- Scrollable if content overflows viewport height

**Responsive behavior:**
- Below `lg` (< 1024px): left panel hidden entirely, right panel fills full screen with 24px horizontal padding
- Mobile (`sm`, < 640px): 16px horizontal padding

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Input internal padding supplemental, label-to-input gap |
| md | 16px | Default form element spacing, card internal padding |
| lg | 24px | Section padding within form, gap between form groups |
| xl | 32px | Gap between wizard steps header and form body |
| 2xl | 48px | Right panel horizontal padding, major vertical rhythm |
| 3xl | 64px | Page-level vertical spacing (not used in auth forms) |

**Exceptions:**
- Touch targets for buttons: minimum 44px height (WCAG 2.1 AA — mobile tap target)
- Wizard step indicator row: 40px height fixed, 8px gap between step dots/labels
- Form submit button: 44px height minimum (`h-11` in Tailwind)

## Typography

All sizes use Inter via `next/font/google`. Language: `pt-BR`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form helper text, descriptions, table cells |
| Label | 14px | 600 (semibold) | 1.4 | Form labels, badge text, navigation items |
| Heading | 20px | 600 (semibold) | 1.3 | Card titles, step titles, section headings |
| Display | 28px | 600 (semibold) | 1.2 | Auth page main heading ("Bem-vindo de volta", "Crie sua conta") |

**Constraint: only 2 weights in components — 400 (regular) and 600 (semibold).** Do not use weight 500 or 700.

**Error text:** 12px, weight 400, color `destructive` (red token). Uses `FormMessage` from shadcn/ui.

**Placeholder text:** 14px, weight 400, color `muted-foreground`.

## Color

Uses shadcn/ui CSS variable system. All values reference CSS variables, not hardcoded hex.

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | Page backgrounds, form panel, input backgrounds |
| Secondary (30%) | `--muted` | `hsl(210 40% 96.1%)` | Wizard step inactive background, disabled state fills, aside backgrounds |
| Accent (10%) | `--primary` | `hsl(221 83% 53%)` | Reserved — see list below |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | Destructive actions and error states only |

**Accent (`--primary`) reserved for:**
1. Primary CTA buttons only ("Criar conta", "Entrar", "Enviar convite")
2. Active wizard step indicator dot and label
3. Focus ring on form inputs (`ring-primary`)
4. Progress bar fill in wizard
5. Link text on auth screens ("Esqueceu a senha?", "Voltar ao login")

**Accent is NOT used for:** card borders, secondary buttons, table headers, badge backgrounds, or passive navigation elements.

**Left brand panel gradient** (decorative only, outside CSS variable system):
- `from-blue-600 to-violet-600` (Tailwind palette values)
- This is the only place hardcoded Tailwind palette colors are permitted in Phase 1

**Role badge colors** (semantic, using shadcn/ui `badge` variant):
| Role | Badge variant | Meaning |
|------|--------------|---------|
| admin | `default` (primary bg) | Full access |
| corretor | `secondary` | Standard access |
| financeiro | `outline` with blue border | Finance access |
| visualizador | `outline` | Read-only |

## Interaction Contracts

### Wizard Step Indicator

- 3 steps, always visible at top of right panel
- Active step: filled circle (primary color), bold label
- Completed step: checkmark icon (Lucide `Check`), muted label
- Inactive step: empty circle (muted border), muted label
- Steps are NOT clickable to navigate backward — use "Voltar" button only
- Progress bar below step indicators fills proportionally: 33% / 66% / 100%

### CNPJ Field Behavior

1. User types raw digits — mask applied automatically: `00.000.000/0000-00`
2. On blur or after 14 digits entered: client-side digit verification runs immediately
3. If valid: show spinner + "Consultando CNPJ..." label
4. On ReceitaWS success: populate "Razão social" field, show inline success message
5. On ReceitaWS failure: show inline error, keep "Razão social" field editable
6. Debounce API call: 500ms after last digit entered

### Password Field Behavior

- Show/hide toggle (Lucide `Eye` / `EyeOff`) on right side of input
- Password strength is NOT shown (Phase 1 scope: minimum 8 chars only)
- Confirm password validation fires on blur, not on keystroke

### Form Submission States

All primary CTA buttons must implement:
- Default: enabled, primary fill
- Loading: disabled + spinner (Lucide `Loader2` with `animate-spin`) + text changes to gerund ("Entrando...", "Criando conta...", "Enviando...")
- Error: button re-enables, error message shown below form (not inside button)
- Success: redirect (no lingering success state on button)

### Toast Notifications (shadcn/ui sonner)

Position: top-right, 4px from edge.

| Event | Type | Copy |
|-------|------|------|
| Invite sent | success | "Convite enviado para [email]" |
| Invite resent | success | "Convite reenviado para [email]" |
| Invite cancelled | info | "Convite cancelado" |
| User removed | info | "[Nome] foi removido da corretora" |
| Session expired | warning | "Sua sessão expirou. Faça login novamente." |
| Generic server error | error | "Ocorreu um erro inesperado. Tente novamente." |

### Role Selection (Invite Flow)

- shadcn/ui `Select` component
- Options listed in order: Administrador, Corretor, Financeiro, Visualizador
- Default selection: Corretor
- Helper text below select: role descriptions shown as static text, NOT tooltip

**Role descriptions shown as static helper text:**
- Administrador: "Acesso total, incluindo configurações e faturamento"
- Corretor: "Gerencia sua própria carteira de clientes e apólices"
- Financeiro: "Visualiza e gerencia contas a receber e a pagar"
- Visualizador: "Apenas visualiza — não cria nem edita registros"

### User Table (User Management)

- Columns: Nome, E-mail, Papel (badge), Status (Ativo / Convite pendente), Ações
- "Ações" column: `DropdownMenu` with options contextual to status
  - Active user: "Alterar papel", "Remover usuário"
  - Pending invite: "Reenviar convite", "Cancelar convite"
- No pagination in Phase 1 (corretoras SMB scope: < 50 users)
- Sortable by: Nome (default A-Z), Papel, Status

## Accessibility Constraints

- All form inputs must have explicit `<label>` association (htmlFor + id pairing)
- Error messages must use `aria-live="polite"` (shadcn/ui FormMessage handles this via Radix)
- Focus must be moved to first error field on form submission failure
- Wizard steps must be announced via `aria-label` on the step indicator region
- Color alone must not convey meaning — role badges use text + color
- Minimum contrast ratio: 4.5:1 for body text, 3:1 for large text (WCAG 2.1 AA)
- Destructive confirmation dialogs trap focus (shadcn/ui Dialog handles via Radix FocusTrap)
