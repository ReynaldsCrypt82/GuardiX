         ```
         let sum = 0
         for (let i = 0; i < len; i++) sum += parseInt(input[i], 10) * (len + 1 - i)
         const remainder = (sum * 10) % 11
         return remainder >= 10 ? 0 : remainder
         ```
      4. `const d1 = calcDigit(digits, 9); const d2 = calcDigit(digits, 10)`
      5. `return parseInt(digits[9],10) === d1 && parseInt(digits[10],10) === d2`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Server Action | Inputs PF/PJ (document, name) chegam como FormData não confiável |
| Server → Postgres | Todas as queries passam por RLS (tenant_id + role + assigned_to) |
| Auth JWT → app_metadata | `tenant_id`, `role` em `app_metadata` (imutável pelo usuário) — NÃO `user_metadata` |
| Service role → DB | `SUPABASE_SERVICE_ROLE_KEY` apenas em Server Actions; nunca exposto via `NEXT_PUBLIC_*` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `public.clients` select policy | mitigate | RLS `tenant_id = (SELECT public.jwt_tenant_id()) AND (role IN admin/fin/vis OR assigned_to = (SELECT auth.uid()))` — wrapping obrigatório para evitar Pitfall 1 |
| T-02-02 | Information Disclosure | Tenant A ↔ Tenant B | mitigate | Toda policy nas 4 novas tabelas usa `tenant_id = (SELECT public.jwt_tenant_id())`; teste `tests/db/rls-clients.test.ts` valida isolamento |
| T-02-03 | Tampering | Input CPF/CNPJ malicioso | mitigate | CHECK constraints `clients_document_length` e `clients_document_digits` no banco; validateCPF/validateCNPJ na Server Action (Wave 1) |
| T-02-04 | Tampering | Soft delete bypass (hard DELETE via SQL) | mitigate | Trigger `public.prevent_hard_delete()` aplicado às 4 novas tabelas em migration 0010 |
| T-02-05 | Information Disclosure | Erro "documento já cadastrado" revela existência em outro tenant | mitigate | Constraint é `UNIQUE (tenant_id, document)` — colisão só ocorre no mesmo tenant (fisicamente impossível vazar inter-tenant) |
| T-02-06 | Spoofing | Criar interação/task com `created_by` forjado | mitigate | Policies `interactions_insert` e `tasks_insert` exigem `created_by = (SELECT auth.uid())` |
| T-02-07 | Elevation of Privilege | Corretor tentando editar cliente de outro corretor | mitigate | Policy `clients_update` exige `(role = 'admin' OR (role='corretor' AND assigned_to = (SELECT auth.uid())))` |
