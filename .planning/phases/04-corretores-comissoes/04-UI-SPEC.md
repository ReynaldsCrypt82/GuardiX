## Typography

All sizes inherit from Phase 1 contract (source: 01-UI-SPEC.md). Only 2 weights used.

| Role | Size | Weight | Line Height | Usage in Phase 4 |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | Table cells, card helper text, form descriptions, notes field |
| Label | 14px | 600 (semibold) | 1.4 | Form labels, stat card titles (`text-sm text-muted-foreground`), column headers |
| Heading | 20px | 600 (semibold) | 1.3 | Card titles (`CardTitle`), section headings (`h2`), tab labels |
| Display | 24px | 600 (semibold) | 1.2 | Page h1 ("Corretores", "Parceiros"), broker dashboard name, stat card metric values (`text-2xl font-semibold`) |

**Constraint: only weights 400 and 600. No weight 500 or 700.**

**BRL metric values** in stat cards: 24px / weight 600 — `text-2xl font-semibold`. Format via `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' })`. Source: 04-CONTEXT.md Claude's Discretion.

**Percentage values** in meta stat card: 24px / weight 600 — `text-2xl font-semibold`. Format: `{value}%`.

**SUSEP number**: 14px / weight 400 — displayed as `text-sm text-muted-foreground` alongside broker name.

**Rate/percentage in table cells**: 14px / weight 400, formatted as `{value * 100}%` (e.g., `0.0500` → `5,00%`). Use `Intl.NumberFormat('pt-BR', { style: 'percent', minimumFractionDigits: 2 })`.

## Color

Uses shadcn/ui CSS variable system (inherited from Phase 1, confirmed in globals.css). No hardcoded hex values in components.

| Role | CSS Variable | Light Mode (oklch) | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | Page backgrounds, form panel, input backgrounds |
| Secondary (30%) | `--card` / `--muted` | `oklch(1 0 0)` / `oklch(0.97 0 0)` | Cards, sidebar, filter bars, table headers |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` (near-black) | Reserved — see list below |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | Destructive actions and error states only |

**Accent (`--primary`) reserved for:**
1. Primary CTA buttons only ("Completar perfil", "Salvar parceiro", "Marcar como pago")
2. Active nav item in sidebar (`bg-primary text-primary-foreground font-semibold`)
3. Focus ring on form inputs (`ring-primary`)
4. Active tab indicator

**Accent is NOT used for:** stat card borders, secondary/outline buttons, table headers, passive badges, filter selects, or breadcrumb links.

## Interaction Contracts

### "Completar perfil de corretor" Flow

This Dialog is triggered from the brokers list when a user with `role='corretor'` exists in `profiles` but has no corresponding `broker_profiles` row.

1. Button "Completar perfil de corretor" appears in the list row actions for users without a broker_profile.
2. Clicking opens a `Dialog` (not a separate page) — mirrors `contemplation-dialog.tsx` pattern.
3. Form uses shadcn/ui Form + Input + Label. No React Hook Form in Phase 4 — use direct FormData + Server Action (same pattern as existing consorcio dialogs).
4. Rate override fields: render 9 Input fields in a 3-column grid (`grid grid-cols-3 gap-3`), one per product type key. Each labeled with the product type name.
5. On submit: show loading state in button, call Server Action, on success close dialog + `revalidatePath` + toast.

### "Marcar comissão como paga" Flow

Trigger: button in apólice detail page (`/[slug]/seguros/[id]`) and quota detail page.

1. Button label: "Marcar comissão como paga". Only visible when `commission_paid_at` is null.
2. Button variant: `outline`, size `sm`.
3. After `commission_paid_at` is set: replace button with a read-only Badge (`bg-green-100 text-green-800 border-green-200`) labeled "Comissão paga" + formatted date.
4. Clicking button opens confirmation Dialog (not inline — uses Dialog primitive).
5. Dialog shows pre-calculated values (broker amount + partner amount if applicable). Admin confirms.
6. On confirm: Server Action inserts `commission_entries` row(s), sets `commission_paid_at` on the source record, `revalidatePath`, toast success.
7. Dialog is NOT re-openable once commission is paid (button replaced by badge).

### Broker Dashboard Month Selector

1. Rendered as a native `<select>` element (matches filter pattern in seguros/page.tsx and consorcio/[id]/page.tsx).
2. Options: last 12 months + current month. Format: "abril 2026" (lowercase month, year). Current month is default.
3. Selecting a month updates the URL via form GET submit (no client-side state) — `?month=2026-04`.
4. Server Component re-fetches commission_entries filtered by `reference_month`.

### Ledger Table (Relatório tab)

1. shadcn/ui `Table` component — same pattern as policy-table.tsx.
2. Columns: Data, Tipo (Badge), Referência (link to policy/quota), Valor (BRL right-aligned), Taxa, Notas.
3. No pagination in Phase 4 (ledger filtered per month per corretor — bounded data set).
4. Tipo badge colors: see Color section (comissao=green, estorno=red, correcao=blue).
5. Valor column: right-aligned (`text-right`), red text for negative amounts (estorno): `text-destructive`.

### Broker List Table

1. Columns: Nome, SUSEP, Meta mensal, Produção do mês, Ações.
2. "Produção do mês" column: count of policies where `assigned_to = broker_id` in current calendar month. Display as `{N} apólice(s)`.
3. "Meta mensal" column: formatted as BRL.
4. "SUSEP" column: display raw value or "—" if null.
5. Ações column: `DropdownMenu` with "Ver dashboard" (link) + "Editar perfil" (opens Dialog).
6. Pagination: 25 per page — same URL pattern as seguros/page.tsx (`?page=N`). Source: 04-CONTEXT.md Claude's Discretion.
7. No filter row on corretores list (small data set; admins see all corretores of their tenant).

### Partner List Table

1. Columns: Nome, CNPJ, E-mail de contato, Taxa padrão, Ações.
2. "Taxa padrão" column: formatted as percentage.
3. Ações column: `DropdownMenu` with "Editar" + "Excluir".
4. Pagination: 25 per page.
5. No filter row.

### Form Submission States (inherited from Phase 1 contract)

All Dialog/form submit buttons implement:
- Default: enabled, primary fill
- Loading: disabled + `Loader2` icon with `animate-spin` + gerund label ("Salvando...", "Cadastrando...", "Confirmando...", "Registrando...")
- Error: button re-enables, error toast via sonner
- Success: Dialog closes, success toast, `revalidatePath`

## Accessibility Constraints

Inherited from Phase 1 contract. Applicable to Phase 4:

- All Dialog form inputs must have explicit `<label>` with `htmlFor` + `id` pairing.
- Error messages use `aria-live="polite"` via shadcn/ui FormMessage or sonner toast.
- Destructive confirmation dialog traps focus (shadcn/ui Dialog via Radix FocusTrap).
- Ledger table: negative amount rows (estorno) use `text-destructive` class — color + sign together convey meaning (not color alone).
- Commission paid badge must include accessible text (not icon-only).
- Minimum contrast: 4.5:1 body text, 3:1 large text (WCAG 2.1 AA).
- Touch targets: 44px minimum for all action buttons.
